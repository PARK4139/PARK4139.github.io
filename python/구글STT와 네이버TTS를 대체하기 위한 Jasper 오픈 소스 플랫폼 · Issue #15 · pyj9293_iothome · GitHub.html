오래 걸렸습니다.. Jasper 셋팅 결과. 현재 아주 잘 작동합니다.

먼저 Jaspser 설치는 전혀 문제가 없었지만 Jasper 특성 상 STT와 TTS 엔진을 선택할 수 있게 해둔 부분에서 Default 엔진 설치 과정에 문제가 있었습니다.

STT는 Default 엔진인 PocketSphinx STT 엔진설치를 생략하고 구글 STT로 설정하여 해결하였고, 정상적으로 구동이 되었습니다.

여기서 STT 인식 후에 TTS를 통한 응답이 전혀 이루어지지 않아 Debug 모드를 통해 mp3를 재생하는 aplay에 문제가 있다는 것을 발견하였습니다. 라즈비안을 위한 사운드카드 수동 선택 명령어가 문제였고, 해당 부분을 삭제하여 해결하였습니다.

결론적으로 구글 STT & eSpeak TTS 엔진을 통해 Jasper 운용이 가능했고 기본적으로 탑재되어 있는 시간 묻기 기능 정도를 수행할 수 있었습니다.

  • 참고로 구글 STT는 이전에 사용하던 구글 Cloud Speach API 가 아닌
    https://groups.google.com/a/chromium.org/forum/#!forum/chromium-dev 를 가입해야 구글 API 콘솔에서 검색이 가능한 구글 STT API 을 사용합니다. 그렇기 때문에 인식률이 이전보다 낮다는 점과 함께 Jasper에서는 Cloud STT를 사용할 수 없다고 합니다.

이제 Jasper에 기본 탑재 되어있는 기능(Modules)를 분석해서 추가적인 기능을 부가하기 위한 부분과 이에 따른 클래스다이어그램 작성을 할 예정입니다.
